## Conversion Architecture

**Page structure (above the fold first):**
1. Hero: headline (pain/desire) + subheadline (mechanism) + primary CTA
2. Social proof: logos, numbers, or a single strong testimonial
3. Problem amplification: 3 bullets, present tense, reader-centric
4. Solution reveal: product/service positioned as the obvious answer
5. Features → Benefits: every feature translated to "which means you..."
6. Testimonials: real names, photo, specific results (not vague praise)
7. Objection handling: FAQ format, pre-empt top 3 objections
8. Final CTA: repeat primary CTA with urgency or scarcity element
9. Footer: minimal — trust badges, ÁSZF link, GDPR notice

## Hungarian Market Rules
- Formal `Ön` for B2B, informal `te` for B2C/younger audience — decide before writing
- Price anchoring: always show value before price (ROI framing)
- GDPR consent checkbox mandatory on any lead form
- Include Hungarian trust signals: Hungarian company reg. number, physical address
- Payment: Barion or Simplepay logos increase trust for Hungarian buyers

## CTA Rules
- Primary CTA: action verb + benefit ("Töltsd le ingyen" not "Submit")
- Button color: high contrast against background (not red — reads as error)
- Single primary CTA per screen — no choice paralysis

## Output Expectations
- Structured page outline with all 9 sections
- Suggested headline/subheadline copy for each section
- CTA text recommendations (3 variations)
- Elementor implementation notes if WP build follows

## Quality Criteria
- Zero navigation links that lead off-page (no distraction)
- Page load < 3s on 4G mobile
- Conversion goal single and clear throughout
