## Wireframe Deliverable

Output a structured wireframe specification in text/ASCII format with annotations, ready for implementation by a developer or Elementor builder.

## Required Sections (in order)

**1. Header / Navigation**
- Logo position (left), nav links count (max 5), primary CTA button
- Sticky behavior: yes/no, background on scroll

**2. Hero Section**
- Layout: full-width / split (50/50) / centered
- Headline character count target (max 8 words for H1)
- Subheadline position, CTA button size and placement
- Visual element: photo / illustration / video / abstract — placeholder note

**3. Trust Bar**
- Logo strip or stat numbers (3-5 items)
- Position: below hero, above fold on desktop

**4. Problem / Solution Block**
- Column count, icon or number treatment
- Copy density: brief (headline + 1 sentence per point)

**5. Social Proof**
- Testimonial card layout, photo size, attribution format
- Count: 3 testimonials minimum

**6. CTA Section**
- Background treatment (contrast with page)
- Headline + subheadline + form or button
- Form fields: list with placeholder text

**7. FAQ**
- Accordion or static
- Count: 5-7 questions

**8. Footer**
- Column count, content per column
- Legal links, GDPR notice placement

## Annotation Requirements
- Every section: spacing notes (top/bottom padding)
- Every interactive element: hover state description
- Mobile: note any section that collapses or reorders on mobile

## Output Format
- ASCII wireframe per section (80-char wide)
- Annotation list below each section
- Total estimated word count for copy

## Quality Criteria
- Wireframe reviewable without design tools (pure text)
- Conversion intent clear at each section (no orphaned content)
