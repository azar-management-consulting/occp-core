## Carousel Architecture

**Optimal length:** 7-10 slides (algorithm favors higher swipe completion rate)
**Aspect ratio:** 1080×1080 (square) or 1080×1350 (portrait, more screen real estate)

## Slide-by-Slide Structure

| Slide | Purpose | Copy Rule |
|-------|---------|-----------|
| 1 (Cover) | Hook — make them swipe | Bold headline, curiosity gap, "→ Swipe" prompt |
| 2 | Problem setup | Validate their experience |
| 3-7 | Content slides | One idea per slide, max 50 words |
| 8-9 | Proof/example | Screenshot, stat, or case result |
| 10 (Last) | CTA slide | Follow + save + link in bio |

## Hook Formulas for Cover Slide
- List format: "5 dolog, ami miatt [desired outcome] meghiúsul"
- Counterintuitive: "Ne [common action] — csináld inkább ezt"
- Secret/insider: "Ezt mondják [audience], akik [achieved result]"
- Before/after: "Egykor [pain state] → ma [result]"

## Copy Rules Per Slide
- One idea, one takeaway per slide (complexity kills swipe-through rate)
- Short sentences: 8-12 words max per line
- Hungarian informal `te` default for IG (unless luxury B2B brand)
- Text position: leave top and bottom 15% clear (safe zone)
- CTA on every 3rd slide (soft): "Mentsd el ezt a posztot"

## Caption Structure
- Line 1: mirror cover slide hook (draws in profile visitors)
- Lines 2-4: expand on value, tease remaining slides
- Line 5: primary CTA ("Mentsd el / Kövesd / Link a bioban")
- Hashtags: 5-8, mix of niche (#vallalkozas) + broad (#marketing), placed at end

## Output Expectations
- 10-slide script (headline + body text per slide)
- Caption (150-200 characters + hashtags)
- Cover slide design notes (color, dominant element, font weight)

## Quality Criteria
- Slide 1 works as a standalone hook without requiring slide 2
- Content provides standalone value (educates/entertains without product pitch until slide 8+)
- Save-worthy: user should want to reference this later
