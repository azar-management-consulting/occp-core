# MCP Tool Catalog

This page is the **authoritative reference** for tools exposed through
[`adapters/mcp_bridge.py`](https://github.com/azar-management-consulting/occp-core/blob/main/adapters/mcp_bridge.py).
Every row here corresponds to a `bridge.register(...)` call in
`build_default_bridge()`.

<Callout type="info">
  External adapters are **env-var gated** — missing credentials simply
  skip the registration block instead of breaking startup.
</Callout>

## Built-in tools (always registered)

| Tool | Namespace | Env vars | Example call | Expected response |
|------|-----------|----------|--------------|-------------------|
| `filesystem.read` | fs | — | `{"path":"notes.md"}` | `{path, content}` |
| `filesystem.write` | fs | — | `{"path":"out.txt","content":"hi"}` | `{path, bytes_written}` |
| `filesystem.list` | fs | — | `{"path":"subdir"}` | `{path, entries[]}` |
| `http.get` | net | — | `{"url":"https://example.com"}` | `{status_code, headers, body}` |
| `http.post` | net | — | `{"url":"...","body":{"k":"v"}}` | `{status_code, body}` |
| `brain.status` | brain | — | `{}` | `{platform, version, bridge, timestamp}` |
| `brain.health` | brain | — | `{}` | API `/health` payload |

<Callout type="warn">
  `filesystem.*` is chrooted to `/tmp/occp-workspace`. Path-escape
  attempts raise `PermissionError` server-side.
</Callout>

## Node execution tools (Tailscale mesh)

| Tool | Namespace | Env vars | Example call | Expected response |
|------|-----------|----------|--------------|-------------------|
| `node.list` | node | — | `{}` | `{nodes[], total}` |
| `node.status` | node | — | `{"node_id":"hetzner-brain"}` | `{node_id, reachable, output, error}` |
| `node.exec` | node | — (SSH key mounted at `/ssh/brain_ed25519`) | `{"node_id":"imac","command":"uptime"}` | `{node_id, command, exit_code, stdout, stderr}` |

<Callout type="warn">
  `node.exec` only accepts commands whose first token is in the
  hard-coded allowlist (`hostname`, `uptime`, `df -h`, `docker ps`,
  `curl`, etc.). Arbitrary execution is structurally blocked.
</Callout>

## External adapter — Supabase (iter-4)

- Source: `adapters/mcp_supabase.py`
- Env vars: `OCCP_SUPABASE_URL`, `OCCP_SUPABASE_SERVICE_ROLE_KEY`
- Read-only by policy — write SQL (`INSERT`, `UPDATE`, `DELETE`, …) is rejected.

| Tool | Example call | Expected response |
|------|--------------|-------------------|
| `supabase.query` | `{"sql":"SELECT id FROM users LIMIT 10"}` | `{status_code, rows}` |
| `supabase.list_tables` | `{}` | `{tables[]}` |
| `supabase.describe_table` | `{"table":"users"}` | `{columns[]}` |

## Calling tools

```python
from adapters.mcp_bridge import build_default_bridge, ToolCall

bridge = build_default_bridge()
result = await bridge.dispatch(ToolCall(
    tool="supabase.query",
    params={"sql": "SELECT count(*) FROM audit_log"},
    agent_id="brain",
))
print(result.status, result.result)
```

Every dispatch runs through the kill switch, the agent tool guard, the
input sanitizer, and a per-call timeout — identical contract for
built-ins and externals.
