# EU AI Act Article 14 ‚Äî Compliance Mapping

**Regulation:** (EU) 2024/1689 ¬∑ **Deadline:** 2026-08-02 (high-risk systems, Art. 113).

Article 14 requires a **3-tier human oversight** architecture:

- **L1 Understand** ‚Äî monitor, detect anomalies, explain decisions (Art. 14(4)(a)-(c))
- **L2 Intervene** ‚Äî disregard / override / reverse model output (Art. 14(4)(d))
- **L3 Halt** ‚Äî one-click STOP bringing the system to a safe state (Art. 14(4)(e))

OCCP's mapping is tracked in [`.planning/EU_AI_ACT_ART14_COMPLIANCE_MAPPING.md`](https://github.com/azar-management-consulting/occp-core/blob/main/.planning/EU_AI_ACT_ART14_COMPLIANCE_MAPPING.md).
This page flattens it into a status table you can audit directly.

## G-6 closure ‚Äî what changed

Before iter-3, only `orchestrator/pipeline.py:40-47` polled the kill
switch. That left three autonomous entry paths unguarded:

- `BrainFlow.process_message()` ‚Äî direct chat ‚Üí agent actions
- `MCPBridge.dispatch()` ‚Äî tool calls from Claude Desktop / Cursor
- `AutoDevOrchestrator.execute()` ‚Äî GitHub webhook ‚Üí PR generation

The fix (single commit per component) adds the guard at the top of
each method:

```python
# top of dispatch / process_message / execute
if _require_ks_inactive is not None:
    _require_ks_inactive()
```

Plus a class-level marker for the compliance test to assert:

```python
class MCPBridge:
    __kill_switch_guarded__ = True
```

The regression test lives at
[`tests/test_eu_ai_act_compliance.py:195-230`](https://github.com/azar-management-consulting/occp-core/blob/main/tests/test_eu_ai_act_compliance.py)
(`test_halt_enforced_across_all_entry_points`) ‚Äî previously `xfail`,
now **PASSING**.

## Art. 14(5) biometric dual-control

**Not applicable** to OCCP today. Art. 14(5) attaches to Annex III
point 1(a) (remote biometric identification); OCCP has no biometric
module (zero matches for `biometric`, `face`, `fingerprint` in the
codebase). A voluntary dual-approver flow for **CRITICAL** actions
(kill-switch deactivation after SECURITY trigger, immutable-file edits,
prod deploy after canary rollback) is tracked under P1.
