Retro-Terminal Aesthetics: The Visual Identity of OCCP

OCCP is a cutting-edge AI orchestration platform wrapped in an interface that evokes 1980s terminals. This is not nostalgia for its own sake. The retro-terminal aesthetic was chosen because it communicates the exact qualities an AI governance tool needs: precision, control, and engineering discipline.

The Psychology of Retro Design in Technology

When users encounter a new technology product, they form trust judgments within seconds. These judgments are largely visual. A pastel-colored, rounded-corner interface signals "consumer friendly, easy to use." A dark, monospace, terminal-inspired interface signals "precision engineering tool, built for professionals." Neither is objectively better -- they attract different audiences and set different expectations.

For an AI governance platform, the expectations must be precise. OCCP manages autonomous AI agents that make decisions with real consequences. The visual language must communicate that every action is monitored, every decision is auditable, and the operator is in control. The retro-terminal aesthetic encodes these values:

  • Dark backgrounds reduce visual noise and focus attention on data. They also evoke the control room environments where critical systems are monitored.
  • Monospace typography signals that every character matters. There is no variable-width ambiguity. Code, IDs, hashes, and timestamps are displayed with the same precision they carry in the system.
  • Green-on-black text is the universal visual shorthand for "computer terminal." It immediately positions OCCP as a systems tool, not a consumer product.
  • Minimal animation communicates stability. The few animations that exist (cursor blink, stage pulse) carry meaning rather than decoration.
Trust in AI governance tools is earned through perceived competence. The retro-terminal aesthetic leverages decades of cultural association between terminal interfaces and engineering excellence. It says: this tool was built by people who understand systems at a fundamental level.

The CRT Scanline Overlay

The most distinctive visual element in OCCP's design is the CRT scanline effect. A semi-transparent overlay creates faint horizontal lines across the entire viewport, simulating the appearance of a cathode ray tube display:

body::before {
  content: '';
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: repeating-linear-gradient(
    0deg,
    transparent,
    transparent 2px,
    rgba(0, 0, 0, 0.04) 2px,
    rgba(0, 0, 0, 0.04) 4px
  );
  pointer-events: none;
  z-index: 9999;
}

The scanlines are 2px transparent bands alternating with 2px bands at 4% opacity. This ratio makes them visible on close inspection but imperceptible during normal reading. The effect is subliminal rather than distracting. The pointer-events: none ensures the overlay never interferes with mouse clicks or touch events. The z-index: 9999 places it above all content, including modals and dropdowns.

The Vignette Effect

A second pseudo-element creates a radial gradient that darkens the viewport edges:

body::after {
  content: '';
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  background: radial-gradient(
    ellipse at center,
    transparent 60%,
    rgba(0, 0, 0, 0.4) 100%
  );
  pointer-events: none;
  z-index: 9998;
}

The vignette serves a functional purpose beyond aesthetics: it naturally draws the eye toward the center of the viewport, where the primary content lives. In the dashboard, this means the pipeline status, task list, and agent panel receive more visual attention than the navigation and footer. The 60% transparent center ensures the effect never obscures content -- it only reduces the brightness of the peripheral areas.

Typography: IBM Plex Mono

OCCP uses IBM Plex Mono as its sole typeface. The font was chosen for several technical and aesthetic reasons:

  • Monospace alignment. Every character occupies the same horizontal width, making tables, code blocks, and hash values perfectly aligned without CSS intervention.
  • Seven weight variants (300-700) provide typographic hierarchy within a single family. Headers use 700, body text uses 400, secondary text uses 300. This creates visual distinction without introducing a second typeface.
  • Designed for screens. IBM Plex Mono was engineered for digital display at small sizes. At 15px body text (OCCP's default), characters remain crisp and legible on both retina and standard displays.
  • Open source license. SIL Open Font License allows unrestricted use, modification, and distribution. No licensing concerns for self-hosted deployments.
:root {
  --font: 'IBM Plex Mono', 'Courier New', 'Lucida Console', monospace;
}

body {
  font-family: var(--font);
  font-size: 15px;
  line-height: 1.75;
  -webkit-font-smoothing: antialiased;
}

The fallback chain includes Courier New and Lucida Console, both monospace fonts available on every major operating system. If IBM Plex Mono fails to load from Google Fonts (network issues, corporate firewalls blocking third-party resources), the design degrades gracefully to a system monospace font.

The Color Palette

The OCCP color system uses exactly 10 named colors. Four carry semantic meaning, three are grayscale for text hierarchy, and three are accent colors for categories and tags:

/* Backgrounds */
--bg:          #0a0a0a    /* Primary background - near black */
--bg-card:     #111111    /* Card/panel backgrounds */
--bg-terminal: #0d1117    /* Code blocks, terminal areas */

/* Semantic colors */
--green:       #33ff33    /* Success, active, primary accent */
--amber:       #ffb000    /* Warning, in-progress */
--red:         #f87171    /* Error, failed, blocked */
--blue:        #60a5fa    /* Links, information */

/* Text hierarchy */
--text:        #e5e5e5    /* Primary readable text */
--text-sec:    #9ca3af    /* Secondary, less important */
--text-muted:  #6b7280    /* Deemphasized, timestamps */

Green: The Primary Accent

Green (#33ff33) is the dominant accent color. It appears in the logo, headings, active states, success indicators, and the cursor animation. This particular shade was chosen because it matches the phosphor green of VT100 terminals -- the most iconic terminal display in computing history. The color carries cultural weight: when a developer sees #33ff33 on a dark background, they instinctively read "terminal."

Glow Effects

Interactive elements use box-shadow glow effects to simulate phosphor bloom -- the slight light bleed around bright text on CRT displays:

.cursor {
  display: inline-block;
  width: 0.6em;
  height: 1.1em;
  background: var(--green);
  vertical-align: text-bottom;
  animation: blink 1s step-end infinite;
  box-shadow: 0 0 6px var(--green);
}

.article-card:hover {
  border-color: var(--green-dim);
  box-shadow: 0 0 12px var(--green-glow);
  transform: translateY(-2px);
}

The glow uses a semi-transparent green (rgba(51, 255, 51, 0.15)) to prevent it from overpowering the content. On hover, the glow intensifies slightly, providing a tactile feedback sensation without breaking the overall visual restraint.

The Design System: Components

Card Components

The article card is the primary content container. It uses a dark background (#111111) that is slightly lighter than the page background (#0a0a0a), creating depth through subtle contrast rather than shadows or borders. The 1px border at #222222 provides definition without visual weight.

Tag Badges

Tag badges use a consistent pattern: 10% opacity background color, full opacity text, and a 30% opacity border. This creates readable, color-coded labels without the visual heaviness of solid backgrounds:

.tag-milestone  { background: rgba(51,255,51,0.1);
                   color: #33ff33;
                   border: 1px solid #1a8c1a; }
.tag-security   { background: rgba(248,113,113,0.1);
                   color: #f87171;
                   border: 1px solid rgba(248,113,113,0.3); }
.tag-architecture { background: rgba(96,165,250,0.1);
                   color: #60a5fa;
                   border: 1px solid rgba(96,165,250,0.3); }

How Nostalgia Builds Trust

The retro-terminal aesthetic works because it bridges the familiar and the novel. AI orchestration is new, unfamiliar, and potentially threatening to operators who are accountable for system behavior. The terminal aesthetic is old, familiar, and associated with systems that operators have controlled successfully for decades.

This bridge is psychological scaffolding. The operator's unconscious assessment goes: "This looks like a tool I know how to use. The people who built it understand the systems I work with. I can trust that the information displayed here is accurate and the controls will behave as expected." The aesthetic does not make the platform more trustworthy -- it makes the operator more receptive to trusting it.

The effect compounds over time. As operators use the dashboard and verify that the information is accurate and the controls are reliable, the initial aesthetic trust is replaced by experiential trust. The visual design served its purpose: it lowered the barrier to initial adoption.

  • Retro-terminal aesthetics communicate precision, control, and engineering authority
  • CRT scanlines at 4% opacity are subliminal, never distracting from content
  • Vignette effect functionally directs attention toward the center of the viewport

  • The 10-color system with 4 semantic colors enables instant state comprehension
  • Nostalgia serves as psychological scaffolding for trust in unfamiliar AI systems
